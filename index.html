React on Rails workshop

In this workshop, we will learn about ReactJS and using ReactJS with a Ruby on Rails.

We will be building an app with React and Rails to learn about React as well as how to integrate it effectively with Rails.

Content

  • React basics.
  • React rendering model.
  • Component LifeCycle methods.
  • Data flow within React components.
  • Animation.
  • Setting up Rails app to use React.
  • CRUD operations on data.
  • Nested data and CRUD operations.
  • Passing Active Record Data to and fro React components.
  • Strategies to use React with Rails like webpack, browserify, etc.
  • Deploying Rails apps with React.

At the end of the workshop, the attendees will get understanding of basics of React and how to integrate it with Rails. They will also have a working example of such app to continue learning further.

Prerequisites

  • Familarity with Rails is expected. No prior knoweldge of React is required.
  • Attendees are expected to bring laptos setup with Rails.

How to deploy a Rails app using Chef?

Motivation - why automate infrastructure?

  • Laziness, impatience, hubris - Larry Wall
  • Why is Chef needed?
  • What do we gain?
  • http://devtoprod.com/courses/chef-fundamentals/sections/chef-why-automate-infrastructure

Write your first recipe

  • Get familiar with Chef directory structure
  • Create a new Chef project
    • Use knife to create a new project
  • Understand tools and terminology
  • Run it on vagrant
    • Recipe for ntp daemon, make sure process is running on Vagrant

How to organize your Chef code?

  • What are Cookbooks?
  • Roles, Attributes, Databags (data in general)
  • Separation of code and data - why this is important
    • Allows reuse of same code across many contexts and envs

How to harness the Chef ecosystem / community

  • Add a few existing cookbooks in the mix
    • Understand how cookbooks are configured using data
    • Start with something basic like apt-cookbook, move on to setting up apt, users, sudoers, ssh
    • Baseline system configuration is done
    • Talk about what baseline means - common to ALL machines.
    • Baseline is a ROLE.
    • Refer back to advantages mentioned in motivation
  • Berkshelf
  • How to tell whether a community cookbook is good quality?
  • How to make changes to cookbooks - fork and use? Wrap in your own?
  • Bonus: contributing back to the community

Start with setting up Rails

  • Install Ruby using cookbook
    • Which version?
    • How?
    • Binary or source?
    • Should it be a part of the baseline role? Maybe, maybe not? Talk about why.
  • Install Bundler using cookbook
    • Same questions as above.

Moving to a n-tier architecture

  • Lets start with 3 tiers - web, app, db
  • DB
    • Setup Postgres server with sane and secure defaults
  • App layer with nginx-passenger
  • Web layer with nginx
  • Launch it on localhost and see that it works

Overarching concerns

  • Always be security conscious
  • Performance tuning
  • Configur-ability - what do I need to do to change a config across 20 machines?

Deployment to AWS, etc.

  • If time permits, our code should be reusable enough to just deploy to AWS/DO.
  • Makes the point about why automate infrastructure clearer.
  • More bonus discussions
  • Versioning infrastructure
  • Immutable infra
  • Docker
  • Testing infra code - continuous integration.
